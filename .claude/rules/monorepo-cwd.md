# Monorepo CWD

A PreToolUse hook auto-prepends `cd $PROJECT_DIR &&` to every Bash command,
ensuring commands run from the repo root. Work with it, not against it.

## Rules
- Never `cd` into subdirectories — the hook handles cwd
- Use root-level `bun run <script>` for package-specific commands (root package.json delegates via `--cwd`)
- For git operations, use repo-root-relative paths: `git add apps/mcp/src/foo.ts`, not `git add src/foo.ts`
- If a command must bypass the hook (e.g. `git -C /absolute/path`), prefix with `NO_CD=1`:
  `NO_CD=1 git -C /some/worktree status`

## Worktrees

In worktrees, `PROJECT_DIR` points to the **worktree root**, not the main repo checkout.
The `.env` file in a worktree is generated by `setupWorktree` and is worktree-specific —
it is not shared with the main checkout or other worktrees.
